Type Checking

  • Static
  • Dynamic

Using a 'Type' to assert a value is its 'instance'.

 

With static type-checking many errors are found before the program runs.

With dynamic type-checking many errors are caught during unit-testing before the program is put in production.

For dynamic type-checking to work together with unit-tests, you must put "dynamic type-declarations" into your code.

UNLIKE with static type-checking, you can choose to add dynamic type-declarations where and when needed.
